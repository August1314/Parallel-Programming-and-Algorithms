# Lab5 设计规格

日期：2026-04-30

## 1. 目标

在 `lab/lab5` 下建立一个独立实验模块，覆盖以下两条实验主线：

1. 基于 OpenMP 的通用矩阵乘法，并比较 `default`、`schedule(static,1)`、`schedule(dynamic,1)` 三种调度策略。
2. 基于 Pthreads 实现 `parallel_for` 动态链接库，在 Linux 环境生成 `libparallel_for.so`，并用它驱动矩阵乘法做正确性与性能验证。

最终交付需形成闭环：

- 核心 C++ 实现
- Docker 统一 Linux 构建与运行链路
- benchmark 数据采集
- 自动绘图
- 表格导出
- 基础自动化测试
- 报告骨架

## 2. 环境约束

- 当前宿主机为 macOS，但 `parallel_for` 题目要求 Linux `.so`。
- `lab5` 的 canonical 构建、运行和 benchmark 统一走 Docker Linux 容器。
- 不修改 `lab0` 到 `lab4` 的现有实现。

## 4. 核心实现设计

### 4.1 公共矩阵协议

`src/matmul_common.hpp` 负责：

- 解析参数：`<m> <n> <k> <seed> <threads> [--dump]`
- 固定 seed `20250401`
- 随机矩阵生成
- `checksum` / `max_abs` 计算
- 小规模矩阵打印
- 稳定 `key=value` 输出

### 4.2 OpenMP 版本

只并行矩阵乘法最外层行循环。三种版本共用同一份源码，通过编译期宏切换调度：

- `openmp_default`
- `openmp_static1`
- `openmp_dynamic1`

输出字段：

- `experiment=matmul`
- `backend=openmp`
- `version=<具体版本>`

### 4.3 `parallel_for` 动态库

固定接口：

```cpp
parallel_for(int start, int end, int inc,
             void *(*functor)(int, void*),
             void *arg,
             int num_threads)
```

本次实现边界：

- 仅支持 `inc > 0`
- 仅实现连续块划分
- 库内部负责线程创建、任务划分和 `join`
- 回调以“单个索引”为粒度执行

动态库产物：

- `bin/libparallel_for.so`

### 4.4 `parallel_for` 矩阵乘法

`src/parallel_for_matmul.cpp` 仅将最外层行循环改造为 `parallel_for` 调用。

输出字段：

- `experiment=matmul`
- `backend=pthreads_parallel_for`
- `version=parallel_for_row_block`

## 5. 构建与运行设计

- Docker 镜像提供 Linux 下的 `g++`、`make` 和 OpenMP 运行时。
- `Makefile` 只描述 Linux 容器内的真实构建规则。
- `run_openmp_matmul.sh` 和 `run_parallel_for_matmul.sh` 在容器内执行已构建好的二进制。
- `parallel_for_matmul` 使用 `rpath=$ORIGIN` 链接，保证运行时能在同目录发现 `libparallel_for.so`。

## 6. 输出契约

所有核心程序保持稳定 `key=value` 输出，至少包含：

- `experiment`
- `backend`
- `version`
- `m`
- `n`
- `k`
- `seed`
- `threads`
- `time_sec`
- `checksum`
- `max_abs`

## 7. Benchmark 设计

固定实验维度：

- 版本：4 个
- 线程数：`1,2,3,4,5,6,7,8`
- 矩阵规模：`512,1024,2048`
- 重复次数：默认 `3`
- seed：`20250401`

输出文件：

- `results/benchmark_results.json`
- `results/summary_by_version.csv`
- `results/performance_table.csv`

状态字段至少区分：

- `ok`
- `failed`
- `partial_failure`
- `checksum_mismatch`
- `baseline_checksum_mismatch`
