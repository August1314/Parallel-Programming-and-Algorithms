# Lab1 实验报告填写要点

## 实验目的

本实验实现串行矩阵乘法，并通过多种实现方式对性能进行对比分析。重点观察基础实现、循环顺序优化、编译优化、循环展开以及高性能数学库对运行时间和浮点性能的影响。

## 实验过程和核心代码

### 版本 1：Python

- 使用纯 Python 三重循环实现矩阵乘法。
- 作为绝对基线，优点是实现直观，缺点是解释器开销大、性能最低。

### 版本 2：C/C++ 基础版

- 使用 C++ 按照最直接的 `i-j-p` 三重循环实现矩阵乘法。
- 相较 Python，编译型语言显著降低了运行时开销。

### 版本 3：调整循环顺序

- 将循环顺序改为 `i-p-j`。
- 这样访问矩阵 `B` 和 `C` 时更接近顺序访问，有利于缓存局部性，通常能进一步提升性能。

### 版本 4：编译优化

- 在版本 3 的实现上启用 `-O3` 编译优化。
- 编译器会进行更积极的指令优化、寄存器分配和循环层面的优化。

### 版本 5：循环展开

- 在版本 4 基础上对内层循环做 4 路展开。
- 这样可以减少循环控制指令带来的额外开销，并为编译器生成更高效指令提供空间。

### 版本 6：Intel MKL

- 使用 `cblas_dgemm` 调用 MKL 的矩阵乘法实现。
- 当前机器无 MKL 环境，因此该版本代码已实现，但需要在 Ubuntu/Intel + MKL 环境中补跑实验数据。

## 实验结果分析

可围绕以下角度展开：

1. Python 与 C++ 的性能差异主要来自解释执行与编译执行的差别。
2. 调整循环顺序后，缓存命中率通常更高，因此运行时间下降。
3. `-O3` 编译优化往往带来最明显的一次性能提升。
4. 循环展开能继续降低部分开销，但收益通常不如编译优化显著，且与处理器和编译器密切相关。
5. MKL 属于高度优化的数学库，在合适硬件和环境下通常能提供最高性能。

## 实验感想

可结合以下内容撰写：

- 同一算法在不同实现方式下性能差异很大，说明工程实现细节对性能影响明显。
- 缓存友好访问模式对矩阵运算非常关键。
- 编译器优化并不是“可有可无”的附加项，而是性能实验中的核心变量。
- 结果正确性校验不能省略，否则性能对比没有意义。
- MKL 这类工业级库展示了人工优化和底层调优的重要价值。
